# 技能库

<p style="font-size: 1.2em;"><strong>是否依赖 CC 沙盒：是</strong></p>

<p style="font-size: 1.2em;"><strong>是否必须绑定工作区：建议绑定</strong></p>

技能库用于管理 Claude Code 可使用的技能。技能可以理解为一组面向特定任务的操作说明、脚本或资源，例如 Git 工作流、浏览器操作、数据处理、插件生成等。

当工作区启用技能后，Claude Code 在处理任务时可以按需调用这些技能，减少重复提示词，提高复杂任务的稳定性。

::: tip 适合什么时候使用
如果您经常让 AI 完成某一类固定任务，例如维护仓库、生成插件、分析网页、处理资源文件，可以把对应能力整理成技能并放入技能库。
:::

## 技能来源

技能库通常包含以下来源：

| 来源 | 说明 |
|---|---|
| 内置技能 | 系统随版本提供的常用技能，适合直接启用 |
| 本地上传 | 从本地导入技能包，适合团队内部沉淀能力 |
| 仓库克隆 | 从 Git 仓库同步技能，适合持续更新的技能集合 |
| 工作区技能 | 绑定到具体工作区，只在对应工作区任务中使用 |

## 基本使用流程

1. 进入 WebUI「工作区」→「技能库」
2. 查看已有技能，确认名称、描述和来源
3. 根据需要上传、克隆或同步技能
4. 将技能关联到目标工作区
5. 在工作区的 Claude Code 任务中使用

技能启用后，不需要每次手动复制说明。Claude Code 会根据任务上下文判断是否需要使用对应技能。

## 上传技能

本地上传适合导入已经打包好的技能。上传前建议检查：

- 技能名称清晰，能表达用途
- 技能说明面向任务目标，而不是实现细节
- 安装步骤不会依赖本机私有路径
- 不包含密钥、Token、Cookie 等敏感信息

上传完成后，可以在技能详情中查看说明，并按需同步到工作区。

## 从仓库克隆技能

如果技能来自 Git 仓库，可以通过仓库地址克隆。适合以下场景：

- 团队统一维护一组技能
- 技能需要随上游更新
- 多个 Nekro Agent 实例共用同一套技能

克隆后如需获取最新版本，可以在技能库中执行同步。同步前建议先确认上游变更可信，避免把不需要的操作说明或脚本带入工作区。

## 工作区同步

技能可以同步到一个或多个工作区。同步后，该工作区中的 Claude Code 任务就能使用这些技能。

建议按工作区职责选择技能：

| 工作区类型 | 推荐技能 |
|---|---|
| 文档项目 | 文档整理、链接检查、翻译辅助 |
| 代码项目 | Git 工作流、测试、代码生成、依赖管理 |
| 运维项目 | 日志分析、配置检查、脚本执行 |
| 内容项目 | 素材处理、网页分析、格式转换 |

不要把所有技能都同步到所有工作区。技能越多，AI 需要判断的上下文也越多，反而可能降低任务聚焦度。

## 自动注入

部分技能可以配置为自动注入。自动注入适合非常稳定、经常需要的基础能力，例如固定的仓库协作流程或团队规范。

使用自动注入时建议遵循：

- 只注入确实长期需要的技能
- 避免多个技能给出互相冲突的操作要求
- 技能说明保持简短明确
- 更新技能后观察几次任务表现

如果发现 Claude Code 在任务中引用了不相关的技能，可以先检查自动注入范围是否过宽。

## 使用建议

- **按任务沉淀**：不要为了“完整”创建技能，优先沉淀高频任务
- **少而精**：一个工作区只同步真正相关的技能
- **先试用再推广**：新技能先在单个工作区验证，再同步到更多工作区
- **定期清理**：长期不用、说明过时或依赖失效的技能应及时移除
- **注意安全**：不要在技能中保存账号密码、访问令牌或内部敏感地址

## 常见问题

### 技能已经同步，为什么任务里没有使用？

技能不是强制执行项。Claude Code 会根据任务判断是否需要使用。如果希望更稳定地触发，可以优化技能名称和描述，让用途更明确。

### 可以把技能当作插件使用吗？

不建议混用。插件是 Nekro Agent 运行时能力，面向聊天、命令、工具和系统集成；技能是 Claude Code 的任务辅助说明，主要服务工作区里的代码、文件和自动化任务。

### 删除技能会影响历史任务吗？

删除后，新任务无法再使用该技能。历史任务记录通常不会因此改变，但如果后续继续同一类任务，可能需要重新选择或同步技能。
