# 资源中心

<p style="font-size: 1.2em;"><strong>是否依赖 CC 沙盒：管理资源不依赖，供 Claude Code 使用时依赖</strong></p>

<p style="font-size: 1.2em;"><strong>是否必须绑定工作区：使用资源时需要挂载到工作区</strong></p>

资源中心用于统一管理工作区会用到的外部资源，例如服务器登录信息、数据库连接、GitHub 凭证、第三方 API Token 等。

当您把资源挂载到某个工作区后，该工作区中的 Claude Code 就可以在执行任务时使用这些资源。您不需要每次都重新发送账号、密码、连接地址或令牌。

## 可以做什么

资源中心适合管理以下内容：

- 服务器 SSH 连接信息
- PostgreSQL、MySQL、Redis 等数据库连接信息
- GitHub CLI 凭证
- 第三方服务 API Token
- 常见后台系统的账号密码
- 需要被多个工作区复用的连接配置

::: tip 适合放到资源中心的情况
如果一组信息会被多个工作区反复使用，或者包含多个字段，例如主机、端口、用户名、密码，就适合放到资源中心统一管理。
:::

::: warning 注意
资源中心可以保存敏感凭证。请只填写您确认可以交给对应工作区使用的信息，并避免把高权限凭证挂载到不需要的工作区。
:::

## 入口位置

资源中心有两个入口：

| 入口 | 用途 |
|---|---|
| 左侧菜单「工作区」→「资源中心」 | 创建、编辑、删除全局资源 |
| 工作区详情页「资源挂载」标签页 | 为当前工作区挂载或卸载资源 |

在「资源中心」里创建资源后，它只是进入资源库。只有挂载到工作区后，Claude Code 才能在该工作区中使用它。

## 创建资源

进入「工作区」→「资源中心」，点击「新建资源」。

您可以直接填写资源，也可以先选择初始化模板。

创建资源时主要填写以下内容：

| 配置项 | 怎么填写 |
|---|---|
| 资源名称 | 写清楚这是什么资源，例如 `生产服务器 SSH`、`GitHub 机器人账号` |
| 资源标签 | 用逗号分隔，例如 `ssh, deploy, github`，方便搜索和分类 |
| 资源备注 | 给管理员看的说明，例如用途、所属项目、适用范围 |
| 资源提示 | 告诉 Claude Code 什么时候可以使用这条资源，以及使用时要注意什么 |
| 字段配置 | 填写实际连接信息或凭证，例如主机、端口、用户名、密码、Token |

资源提示建议写得明确一些，例如：

- 「用于查看部署日志和执行只读巡检」
- 「仅用于查询数据库状态，除非明确要求，不要修改或删除数据」
- 「用于 GitHub PR、Issue 和 Actions 排查」

## 使用初始化模板

新建资源时可以点击「选择模板」，快速套用常见资源配置。

当前可选模板包括：

- SSH 密钥连接
- SSH 密码连接
- Root 密码连接
- PostgreSQL 连接
- MySQL 连接
- Redis 连接
- GitHub CLI 凭证
- 通用令牌凭证
- 账号密码凭证

模板会自动填充常用字段、标签和提示词。套用后仍然可以继续修改。

## 配置字段

每条资源由一个或多个字段组成。字段就是这条资源需要保存的具体信息。

常见字段示例：

| 资源类型 | 常见字段 |
|---|---|
| SSH 连接 | 主机地址、端口、用户名、密码或私钥 |
| 数据库连接 | 主机地址、端口、数据库名、用户名、密码 |
| GitHub 凭证 | 访问令牌、账号名称 |
| API Token | 访问令牌、服务地址 |
| 账号密码 | 账号、密码、登录入口地址 |

### 字段类型

字段类型用于帮助您整理资源内容。常用类型包括：

- 文本
- 密码/令牌
- 主机地址
- 端口
- 私钥
- 用户名
- 数据库名
- JSON

### 敏感信息

密码、Token、私钥等字段建议开启「敏感信息」。

建议标记为敏感信息的内容：

- 登录密码
- API Token
- SSH 私钥
- 数据库密码
- Cookie 或会话凭证

### 兼容环境变量名

某些工具会默认读取固定名称的变量。您可以在「兼容环境变量名」里填写这些名称，让 Claude Code 更方便地使用对应工具。

常见例子：

| 场景 | 可填写的兼容变量名 |
|---|---|
| GitHub CLI | `GH_TOKEN`, `GITHUB_TOKEN` |
| 自定义 API Token | 按您的工具要求填写，例如 `API_TOKEN` |

::: warning 避免重复
同一个工作区里，不要让多条资源使用相同的兼容环境变量名。挂载时如果存在冲突，系统会提示您处理。
:::

## 挂载资源到工作区

创建资源后，需要把它挂载到工作区才能使用。

操作步骤：

1. 进入目标工作区详情页
2. 打开「资源挂载」标签页
3. 在资源库中找到需要的资源
4. 点击「挂载」

挂载完成后，您就可以在绑定该工作区的频道中委托任务，例如：

- 「使用服务器资源查看最近的部署日志」
- 「用 GitHub 凭证检查这个仓库的 Actions 失败原因」
- 「连接数据库，帮我查看最近 24 小时的订单统计」

## 管理已挂载资源

在工作区「资源挂载」页面中，您可以：

- 查看当前工作区已挂载资源
- 从工作区卸载资源
- 拖动调整资源顺序
- 查看资源标签
- 查看资源被哪些工作区引用

卸载资源只会取消当前工作区的使用权限，不会删除资源本身。

## 删除资源

在全局「资源中心」中可以删除资源。

如果资源已经被工作区挂载，删除前系统会提示受影响的工作区。确认删除后，这些挂载关系也会一并移除。

::: danger 谨慎删除
删除资源不可撤销。删除后，依赖该资源的工作区任务可能无法继续访问对应服务器、数据库或外部服务。
:::

## 常见配置示例

### GitHub CLI 凭证

适合用于让 Claude Code 查看 PR、Issue、Actions，或协助处理 GitHub 仓库任务。

配置建议：

1. 新建资源，选择「GitHub CLI 凭证」模板
2. 填写访问令牌
3. 确认令牌字段开启「敏感信息」
4. 保留 `GH_TOKEN`、`GITHUB_TOKEN` 等兼容变量名
5. 挂载到需要操作 GitHub 的工作区

### 服务器 SSH 连接

适合用于部署检查、日志排查、服务器巡检等任务。

配置建议：

1. 新建资源，选择 SSH 密钥连接或 SSH 密码连接模板
2. 填写主机地址、端口、用户名和认证信息
3. 将密码、私钥、口令短语标记为敏感信息
4. 在资源提示中写明允许的操作范围
5. 挂载到运维或项目工作区

### 数据库连接

适合用于查询数据、检查连接状态、导出结构信息等任务。

配置建议：

1. 新建资源，选择 PostgreSQL、MySQL 或 Redis 模板
2. 填写主机地址、端口、数据库名、用户名和密码
3. 将密码标记为敏感信息
4. 在资源提示中明确默认只读
5. 挂载到数据分析或运维工作区

## 与工作区环境变量的区别

工作区配置页也可以直接设置环境变量，但资源中心更适合管理一组可复用的信息。

| 能力 | 适合场景 |
|---|---|
| 工作区环境变量 | 只属于当前工作区的简单配置 |
| 资源中心 | 可复用、可分类、包含多个字段、需要挂载到多个工作区的资源 |

如果只是某个工作区独有的一两个简单变量，可以放在工作区环境变量中。

如果是服务器、数据库、账号、令牌等可以复用的资源，建议放到资源中心。
