# 2.3.3 版本特性

`2.3.3` 是 Nekro Agent 以工作区为中心的第一个正式版本。

这个版本将 AI 从「聊天工具」升级为「能做事的工作伙伴」：工作区体系让任务有了独立空间，Claude Code 沙盒让 AI 具备了代码执行和文件处理能力，记忆系统让 AI 拥有了长期上下文，MCP 让 AI 可以连接外部工具和数据源。

## 核心能力

### 工作区体系

每个任务都有独立空间。工作区统一管理频道、模型、知识、记忆和工具配置。

- 创建工作区并绑定一个或多个频道
- 每个工作区独立配置 CC 模型组
- 工作区级的提示词、技能和环境变量管理
- 工作区间互不干扰

→ 详见 [工作区总览](/docs/03_workspace/overview)

### Claude Code 沙盒

工作区的核心执行引擎。AI 不再只能聊天，可以真正动手做事。

- 预装 Python 3.13、Node.js 20、Git、GitHub CLI 等完整开发工具链
- 内置浏览器自动化（Playwright + Chromium）
- 文件双向交换：上传文件到沙盒，沙盒产出的文件自动发回
- 异步任务执行：任务后台运行，完成后自动推送结果
- 支持重启、重建和会话重置

→ 详见 [Claude Code 沙盒](/docs/03_workspace/claude_code_sandbox)

### 记忆系统

AI 拥有了长期记忆，越用越懂您。

记忆系统会自动整理工作区中的项目背景、成员偏好、历史决定和任务结论，并在后续对话中按需参考。

- 自动从对话和任务中沉淀长期上下文
- 后续问答中自动回忆相关信息
- 重要信息更容易被保留，不再相关的信息逐渐降低优先级
- 支持手动重建

→ 详见 [记忆系统](/docs/03_workspace/memory_system)

### MCP 服务管理

通过 MCP 协议连接外部工具和数据源，扩展 AI 的能力边界。

- 支持 stdio、SSE、HTTP 三种传输类型
- 工作区级配置，不同工作区使用不同工具
- 卡片式管理界面，直观操作
- 丰富的社区生态：GitHub、数据库、搜索引擎、监控系统等

→ 详见 [MCP 服务管理](/docs/03_workspace/mcp_management)

### CC 模型组

独立于基础模型组的 CC 模型组配置，专为 Claude Code 沙盒场景设计。CC 沙盒的任务复杂度通常高于普通对话，独立配置模型可以为沙盒任务选择更合适的模型。

## 各能力与 CC 沙盒的依赖关系

| 能力 | 是否依赖 CC 沙盒 | 是否必须绑定工作区 |
|---|---|---|
| Claude Code 沙盒 | — | 是 |
| 知识库 | 否 | 是 |
| 记忆系统 | 否 | 是 |
| MCP 服务 | 是 | 是 |
| 定时器 | 否 | 否 |

**不需要一次性配置所有能力。** 建议从创建工作区开始，先启用知识库和记忆系统（无需 CC 沙盒），再逐步配置 CC 沙盒和 MCP 服务。

## 其他改进

- **命令系统重构**：命令注册、解析、分组、权限、补全和前端管理的系统化升级
- **定时器管理**：定时器列表、状态查看和管理操作
- **WebUI 增强**：聊天消息样式、SSE 实时推送、Agent 活动卡片、工作区导航

## 升级建议

1. 从旧版本升级后，先创建一个工作区，绑定一个测试频道
2. 上传一些文档到知识库，启用记忆系统，体验知识和记忆的效果
3. 配置 CC 模型组，启动 CC 沙盒，试一个需要代码执行的任务
4. 如果需要外部工具，配置 MCP 服务
5. 在测试频道验证通过后，再迁移到正式使用的频道
