# 频道管理

频道管理页用来查看和管理每一个聊天频道。

如果您想知道某个频道现在是什么状态、绑定了什么人设、最近消息怎么样、为什么不回复，通常都应该先来这里看。

## 这个页面能做什么

频道管理里通常可以完成这些事情：

- 查看所有频道
- 搜索频道
- 按类型和状态筛选
- 查看频道详情
- 调整频道状态
- 重置当前频道上下文
- 查看实时消息
- 查看覆盖配置
- 查看这个频道下的插件数据

![频道管理列表页](/assets/advanced/channel_management/channel_management_list.png)

## 三种常见频道状态

### 激活

这是正常工作状态。

在这个状态下，频道会按正常规则参与触发和回复。

### 旁观

这个状态会保留频道记录，但不走正常回复流程。

更适合：

- 暂时只看消息，不想让 Agent 回答
- 先观察频道情况
- 调试触发问题

### 停用

这个状态下，频道不会正常参与使用。

如果某个频道暂时不希望被 Agent 接管，可以直接停用。

## 什么时候该切旁观

旁观通常适合下面这些情况：

- 您想先收集消息，不急着让 Agent 回复
- 您怀疑这个频道的触发规则有问题，想先观察
- 您不希望某个频道继续正常对话，但还想留着记录

## 频道详情里能看什么

点进一个频道后，通常最值得先看：

- 当前人设
- 消息数量
- 参与用户数
- 最后活跃时间
- 当前频道状态

![频道详情页](/assets/advanced/channel_management/channel_detail.png)

## 覆盖配置有什么用

覆盖配置只对当前频道生效。

适合：

- 某个频道要单独改一项设置
- 不想影响全局配置
- 想做小范围测试

## 实时消息页适合什么时候看

实时消息页更适合排查：

- 消息有没有进来
- 消息是谁发的
- 回复有没有真正发出去
- 当前频道是不是收得到系统消息或插件输出

## 插件数据页适合什么时候看

如果某个频道的问题和插件有关，可以在插件数据页里查看这个频道下留下来的插件数据。

这更适合：

- 排查插件状态异常
- 看某个插件在这个频道里存了什么
- 删除测试遗留数据

## 使用建议

- 频道不回复时，先看状态是不是切成了旁观或停用
- 如果是单频道问题，优先查覆盖配置
- 如果是插件问题，再看这个频道下的插件数据
