# 仪表盘

仪表盘用于快速了解 Nekro Agent 当前的运行状态。它适合日常巡检、排查异常活跃频道、观察消息趋势和确认服务是否正常。

::: tip 先看仪表盘，再看日志
如果 Bot 突然变慢、变安静或调用量异常，建议先看仪表盘确认整体趋势，再进入日志中心查看细节。
:::

## 可以看到什么

仪表盘通常会展示以下信息：

| 区域 | 用途 |
|---|---|
| 总览卡片 | 查看消息、频道、用户、任务等核心数量 |
| 趋势图 | 观察一段时间内的消息量或调用变化 |
| 排行列表 | 找出最活跃的频道、用户或功能入口 |
| 分布统计 | 了解不同类型消息、适配器或状态的占比 |
| 实时状态 | 查看服务当前是否在线、是否有异常波动 |

不同版本的页面布局可能略有变化，请以 WebUI 实际显示为准。

## 常见使用场景

### Bot 突然不回复

可以按这个顺序检查：

1. 仪表盘是否还有新消息进入
2. 活跃频道是否集中在某几个频道
3. 趋势图是否从某个时间点开始明显下降
4. 再去日志中心查看同一时间段的错误信息

如果仪表盘没有新消息，优先检查适配器连接；如果有消息但没有回复，再检查模型、配额和日志。

### 调用量突然升高

关注排行和趋势：

- 是否某个频道消息量异常高
- 是否某个用户频繁触发 Bot
- 是否最近开启了更宽松的触发规则
- 是否新插件或命令带来了额外调用

必要时可以配合频道管理中的禁用、观察模式或覆盖配置来限制范围。

### 观察版本更新后的状态

更新 Nekro Agent 后，可以通过仪表盘确认：

- 消息趋势是否恢复正常
- 频道和用户数据是否正常显示
- 是否出现异常高的错误或失败趋势
- 工作区、知识库、插件等功能是否仍有活动

## 数据刷新

部分仪表盘数据会自动刷新，部分统计可能按固定时间窗口计算。遇到短时间内数据没有立刻变化时，可以稍等片刻或刷新页面。

如果页面长时间没有更新，但 Bot 实际仍在工作，建议检查 WebUI 与后端服务连接是否正常。
