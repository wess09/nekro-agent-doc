# 插件使用原则

::: warning 安全提示

插件市场中的插件可能来自站外仓库。
在安装或更新前，请自行确认插件来源、代码内容和权限范围。

:::

插件能明显扩展 Nekro Agent 的能力，但插件越多，不代表效果一定越好。
多数情况下，插件管理的重点不是“装满”，而是“装对”。

## 先从少量插件开始

第一次使用时，建议先装少量、用途明确的插件。

更推荐的做法是：

- 先装 1 到 2 个您确实需要的插件
- 先确认它们能稳定工作
- 再逐步补其他能力

## 避免功能重复

不要同时安装多个功能很接近的插件。

这样做最常见的问题是：

- AI 上下文变乱
- 方法太多，调用选择变差
- 明明能做同一件事，却经常选错插件

## 优先看来源和维护状态

在安装插件前，建议至少先看：

- 插件来源是否可信
- 作者是否还在维护
- 文档是否清楚
- 这个插件到底解决什么问题

如果一个插件来源不清、描述模糊、更新方式也不明确，最好先不要装到正式环境。

## 更新插件前先看风险

插件更新不只是“多了新功能”，也可能带来：

- 行为变化
- 配置变化
- 兼容性问题
- 安全风险

如果您正在正式环境使用某个插件，更新前建议先确认：

- 更新内容值不值得
- 是否会影响现有流程
- 是否需要先在测试环境验证

## 什么时候该卸载

如果出现下面这些情况，通常就应该考虑停用或卸载：

- 长期不用
- 功能和别的插件重复
- 插件经常报错
- 插件来源和安全性拿不准

## 使用建议

- 插件越少，排查问题越容易
- 先保证核心插件稳定，再考虑扩展能力
- 对正式环境来说，稳定通常比“功能多”更重要
