# 社区插件

社区插件是由社区作者提供的扩展能力。它们可以让 Nekro Agent 支持更多平台、工具、玩法和自动化场景。

与内置插件相比，社区插件的能力更丰富，但也更需要关注来源、维护状态和权限范围。

## 可以做什么

社区插件常见用途包括：

- 接入第三方服务
- 增加群聊娱乐功能
- 扩展数据查询能力
- 对接外部 API
- 增加特定平台能力
- 定制业务流程
- 扩展角色扮演或互动玩法

::: warning 安全提示
社区插件可能来自站外仓库。安装或更新前，请确认插件来源、作者说明、权限范围和维护状态。
:::

## 在哪里管理

进入 WebUI 的「插件管理」页面，可以查看已经安装的插件，并进行启用、停用、配置和更新。

如果系统提供插件市场或插件仓库入口，建议优先从官方推荐入口安装。

## 选择插件前看什么

安装社区插件前，建议先确认：

| 检查项 | 建议 |
|---|---|
| 插件来源 | 优先选择来源清楚、作者可信的插件 |
| 插件用途 | 确认它确实解决您的需求，不要为了尝鲜安装 |
| 文档说明 | 优先选择配置说明清楚的插件 |
| 维护状态 | 优先选择近期仍在维护的插件 |
| 权限范围 | 关注它是否会访问外部服务、发送消息、管理群聊或处理敏感信息 |

## 安装建议

建议按以下方式安装社区插件：

1. 先阅读插件说明
2. 确认插件适用于当前 Nekro Agent 版本
3. 安装后先在测试频道启用
4. 配置必要参数
5. 发送简单任务验证效果
6. 稳定后再用于正式频道

::: tip 推荐
正式环境不要一次性安装很多社区插件。建议一次只新增一个插件，确认稳定后再继续添加。
:::

## 配置社区插件

不同社区插件的配置项不同。通常需要在插件管理页面中填写：

- API Key
- 服务地址
- 账号或授权信息
- 默认开关
- 频道限制
- 权限或管理员配置

如果插件需要凭证或令牌，建议同时参考 [资源中心](/docs/03_workspace/resource_center)，把可复用的账号、令牌、服务器或数据库信息统一管理。

## 更新插件

更新社区插件前，建议先确认：

- 更新内容是否与当前使用场景有关
- 是否修改了配置项
- 是否可能影响已有功能
- 是否需要先备份配置
- 是否适合先在测试频道验证

如果当前插件运行稳定，而新版本只是增加您暂时用不到的功能，可以不急于更新。

## 什么时候应该停用或卸载

出现以下情况时，建议停用或卸载插件：

- 长期不用
- 与其他插件功能重复
- 经常报错或影响正常对话
- 来源不清楚
- 维护停止且存在兼容问题
- 需要过高权限但收益不明显

## 常见问题

### 插件装得越多越好吗？

不是。插件越多，AI 可选能力越多，也更容易选错工具。建议只保留真正需要的插件。

### 社区插件可以直接上正式环境吗？

不建议。最好先在测试频道启用，确认配置正确、行为符合预期后，再用于正式频道。

### 插件需要密钥怎么办？

如果只是该插件独有配置，可以直接填在插件配置中。如果同一组凭证会被多个工作区或任务复用，建议放到资源中心统一管理。
