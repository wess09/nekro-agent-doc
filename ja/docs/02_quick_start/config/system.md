# システム設定

Nekro Agentは柔軟なシステム設定オプションを提供し、ニーズに応じてシステム動作をカスタマイズできます。このドキュメントでは、基本的なチャット会話機能を完了するために必要な主要なシステム設定項目を紹介します。

## ⚙️ 設定へのアクセス

### WebUIによる設定（推奨）

1. WebUIにアクセスします：`http://<あなたのサーバーIP>:8021`
2. ログイン後、「システム設定」→「基本設定」に移動します

注意：

1. 設定項目を変更する前に、設定項目の上にマウスをホバーして`!`アイコンをクリックし、設定項目の説明を確認してください（利用可能な場合）。これらの説明は、設定項目の使用方法に関する非常に有用な指示を提供することがよくあります
2. 設定の変更は「保存」ボタンをクリックして初めて有効になります

### 設定ファイルによる変更

::: warning 注意
必要でない限り、設定ファイルによるシステム設定の変更は推奨されません。変更を行う必要がある場合は、YAMLの基本構文を完全に理解し、設定項目の意味を深く理解していることを確認してください。そうしないと、システムが正常に動作しない可能性があります
:::

デフォルトのインストールパスでは、設定ファイルは`~/srv/nekro_agent/configs/nekro-agent.yaml`にあります

注意：設定ファイルを直接変更した後、`/conf-reload`コマンドを使用するか、NAサービスを再起動する必要があります

## 📝 概念

### 設定キー

設定キーは、設定ファイル内の設定項目を識別するために使用される一意の識別子で、**大文字の英単語とアンダースコア**で構成されています。WebUIの「システム設定」→「基本設定」ページのすべての設定名の下で確認でき、設定ファイルのインデックスキーと一対一で対応しています。例えば、`ENABLE_NEKRO_CLOUD`はNekroAIクラウドサービス機能を有効にするかどうかを制御する設定キーです。以下の必要な設定項目は`<設定名> - [設定キー]`の形式で表示されます。ページ上で`Ctrl + F`を使用して設定キーを検索し、対応する設定項目を素早く見つけることができます

### 設定値

設定値は、設定キーに対応する値です。WebUIの「システム設定」→「基本設定」ページのすべての設定名の下で確認でき、設定ファイルのフィールド値と一対一で対応しています。例えば、`ENABLE_NEKRO_CLOUD`の設定値が`true`の場合、NekroAIクラウドサービス機能が有効であることを示します

## 🔑 必要な基本設定

「システム設定」→「基本設定」ページで、通常の使用に影響を避けるために、まず以下の重要な設定項目を完了することをお勧めします

### 管理者アカウント - [SUPER_USERS]

管理者アカウントは、システムコマンドの使用を許可されたアカウントを設定するために使用されます。通常、管理者ユーザーのQQアカウントとして設定され、複数のアカウントを設定できます。管理者として設定されたQQユーザーは、`QQアカウント + デフォルトパスワード(123456)`を使用してWebUIにログインし、管理操作を行うことができることに注意してください。ログイン後、速やかにパスワードを変更してください

## 🔒 必要なモデルグループ設定

「システム設定」→「モデルグループ」ページで、少なくとも1つの使用可能なLLM会話モデルグループを設定してください。そうしないと、正常に使用できません

ユーザーの迅速な設定を容易にするため、システムは[Nekro公式リレーステーション](https://api.nekro.ai)からチャット、描画、埋め込みの3つのタイプのモデルグループ設定をプリセットしています。[Nekro公式リレーステーション](https://api.nekro.ai)に直接アクセスしてアカウントを登録し、キーを作成してそれぞれに入力することで使用を開始できます（新規ユーザー登録には1ドルのサイト残金が提供され、一部の低頻度・低コストモデルの使用ニーズを満たすことができます）。他の公式または非公式のOpenAI互換インターフェースモデルチャネルがある場合は、APIアドレスを自分で置き換えてアクセスし使用することができます
