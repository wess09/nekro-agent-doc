# ワークスペースタイマー

<p style="font-size: 1.2em;"><strong>CC サンドボックスへの依存：なし</strong></p>

<p style="font-size: 1.2em;"><strong>ワークスペースへのバインド：不要</strong></p>

タイマーは AI に **自発的に作業** させる機能です。メッセージを送るまで待つのではなくなります。

タイマーを設定すると、AI はスケジュールに従ってタスクを自動実行します。毎朝ニュースサマリーの送信、毎週のプロジェクト週報生成、1時間ごとのシステム状態チェックなどです。タイマーがトリガーされると、AI はタスク説明を理解して自律的に実行します。単に固定スクリプトを実行するだけではありません。

![タイマーリスト](/assets/workspace/timers/timer_list.png)

::: tip 柔軟な使い方
タイマーは Claude Code サンドボックスにもワークスペースへのバインドにも依存しません。タイマー単独でシンプルなリマインダーとして使用することも、ワークスペース内で CC サンドボックスと組み合わせて複雑な自動化フローを実現することもできます。
:::

## タイマーでできること

### 自動リマインダー

- 「毎朝9時にメール確認のリマインダーを送って」
- 「毎週金曜の午後に週報提出のリマインダーを送って」
- 「5分後にメッセージ返信のリマインダーを送って」

### 定期タスク

- 毎日のニュース/情報サマリー
- 毎週のプロジェクト進捗レポート
- 定期的なデータバックアップリマインダー
- 祝日の自動挨拶

### CC サンドボックスと組み合わせた自動化（ワークスペース内）

タイマーがワークスペース内で CC サンドボックスと組み合わせて使用されると、能力がさらに解放されます：

- 毎日指定 Web サイトのコンテンツを自動取得してサマリー生成
- 毎週自動テストを実行してレポート送信
- 毎時監視データをチェックして異常時にアラート
- 定期的なファイル整理とアーカイブ

## タイマータイプ

### ワンショットタイマー

指定時間に1回トリガーされた後に自動完了します。一時的なリマインダーや単発タスクに適しています。

一般的な表現：

- 「5分後にメッセージ返信のリマインダーを送って」
- 「今夜8時にデプロイ結果の確認リマインダーを送って」
- 「来週水曜の午後3時にレポート提出のリマインダーを送って」

### 繰り返しタイマー（Cron）

周期的に繰り返し実行します。長期的な自動化タスクに適しています。

繰り返しタイマーは以下をサポート：

- 標準 cron 式で周期を定義
- 一時停止、再開、即時実行
- 営業日または祝日でトリガー時間をフィルタリング

## タイマーの作成

### 会話から直接作成

最もシンプルな方法——AI に必要な定期タスクを直接伝えるだけです：

- 「毎朝9時にメール確認のリマインダーを送って」
- 「5分後にメッセージ返信のリマインダーを送って」
- 「毎週月曜の午前10時に週報サマリーを生成して」

AI は説明に基づいて適切なタイマーを作成します。作成後、WebUI のタイマーリストで確認・管理できます。

### 開発者向け作成

プラグインを開発している場合は、プラグイン内から定期タスクを作成することもできます。詳細は [プラグイン定期タスク開発](/ja/docs/04_plugin_dev/06_timer_development) を参照してください。

## Cron 式

繰り返しタイマーは標準の5桁 cron 式を使用します：

```
分 時 日 月 曜日
```

一般的な例：

| 式 | 説明 |
|---|---|
| `0 9 * * *` | 毎日午前9:00 |
| `30 8 * * 1-5` | 月曜〜金曜 午前8:30 |
| `0 */2 * * *` | 2時間ごと |
| `0 9 1 * *` | 毎月1日 午前9:00 |
| `0 10 * * 1` | 毎週月曜 午前10:00 |

## 営業日モード

cron 式に加えて、営業日によるフィルタリングが可能です：

| 値 | 説明 |
|---|---|
| `none` | 制限なし。cron のまま通常トリガー |
| `mon_fri` | 月曜〜金曜のみ |
| `weekend` | 週末のみ |
| `cn_workday` | 中国法定営業日のみ（振替出勤日を含む） |
| `cn_restday` | 中国法定休日のみ（振替休日を含む） |

## タイマーの管理

WebUI のタイマーリストで、既存のタイマーに対して以下の操作が可能です：

- **詳細の確認**：cron 式、次回トリガー時間、バインドされたチャンネル
- **即時実行**：手動で1回トリガー
- **一時停止 / 再開**：自動トリガーの一時停止または再開
- **削除**：永続的に削除

::: info WebUI の機能範囲
現在のバージョンの WebUI はタイマーの閲覧と管理機能を提供しています。タイマーの作成は AI の対話またはプラグインコードで行われます。
:::

## トリガー動作

タイマーがトリガーされると、システムはイベント説明を AI にプッシュし、AI がコンテキストを理解した上で自律的に実行を判断します。つまり：

- タイマーの動作は固定スクリプトではなく、AI がイベント説明と現在のコンテキストに基づいてインテリジェントに実行する
- イベント説明が具体的であるほど、AI の実行効果が良い
- ワークスペースでメモリシステムと CC サンドボックスが有効な場合、AI は長期メモリとコード実行能力を活用してより複雑な定期タスクを完了できる

## 使用のヒント

- **シンプルに始める**：まず毎日のリマインダーを1つ試して、タイマーの効果を体験
- **具体的に記述する**：「毎朝9時に今日の TODO をリマインダーして」は「毎日リマインダーして」よりはるかに効果的
- **ワークスペースと組み合わせる**：タイマーをワークスペースにバインドし、ナレッジベースやメモリシステムと連携させると、AI が定期タスク実行時により豊富なコンテキストを持てる
- **営業日モードを活用する**：中国法定営業日モードは振替処理に特に便利
