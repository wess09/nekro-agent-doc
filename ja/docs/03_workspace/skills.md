# スキルライブラリ

<p style="font-size: 1.2em;"><strong>CC サンドボックスへの依存：あり</strong></p>

<p style="font-size: 1.2em;"><strong>ワークスペースへのバインド：推奨</strong></p>

スキルライブラリは Claude Code が使用できるスキルを管理するためのものです。スキルとは、特定タスク向けの操作手順、スクリプト、リソースのセットです。例えば Git ワークフロー、ブラウザ操作、データ処理、プラグイン生成などです。

ワークスペースでスキルを有効にすると、Claude Code はタスク処理時に必要に応じてこれらのスキルを呼び出し、繰り返しのプロンプトを削減し、複雑なタスクの安定性を向上させます。

::: tip こんな時に使いましょう
AI に特定の定型タスクを頻繁に依頼する場合、例えばリポジトリ管理、プラグイン生成、Web ページ分析、リソースファイル処理など、対応する能力をスキルとして整理してスキルライブラリに入れましょう。
:::

## スキルのソース

スキルライブラリには通常、以下のソースが含まれます：

| ソース | 説明 |
|---|---|
| 組み込みスキル | システムがバージョンに同梱するよく使うスキル。そのまま有効化可能 |
| ローカルアップロード | ローカルからスキルパッケージをインポート。チーム内での能力蓄積に適する |
| リポジトリクローン | Git リポジトリからスキルを同期。継続的に更新されるスキルセットに適する |
| ワークスペーススキル | 特定のワークスペースにバインドされ、対応するワークスペースのタスクでのみ使用 |

## 基本的な使用フロー

1. WebUI の「ワークスペース」→「スキルライブラリ」に入る
2. 既存のスキルを確認し、名前、説明、ソースを確認
3. 必要に応じてスキルをアップロード、クローン、または同期
4. スキルを対象ワークスペースに関連付け
5. ワークスペースの Claude Code タスクで使用

スキルを有効にした後、毎回手動で説明をコピーする必要はありません。Claude Code はタスクのコンテキストに基づいて、対応するスキルを使用するかどうかを判断します。

## スキルのアップロード

ローカルアップロードは、すでにパッケージ化されたスキルのインポートに適しています。アップロード前に以下を確認することをお勧めします：

- スキル名が明確で、用途を表現していること
- スキルの説明がタスク目標に向けたもので、実装詳細ではないこと
- インストール手順がローカルのプライベートパスに依存しないこと
- キー、Token、Cookie などの機密情報を含まないこと

アップロード完了後、スキル詳細で説明を確認し、必要に応じてワークスペースに同期できます。

## リポジトリからのスキルクローン

スキルが Git リポジトリ由来の場合、リポジトリアドレスからクローンできます。以下のようなシーンに適しています：

- チームでスキルを統一管理
- スキルをアップストリームの更新に合わせたい
- 複数の Nekro Agent インスタンスで同じスキルセットを共有

クローン後に最新バージョンを取得する場合は、スキルライブラリで同期を実行できます。同期前にアップストリームの変更が信頼できることを確認し、不要な操作手順やスクリプトがワークスペースに入り込むのを避けてください。

## ワークスペースへの同期

スキルは1つまたは複数のワークスペースに同期できます。同期後、そのワークスペースの Claude Code タスクでこれらのスキルを使用できるようになります。

ワークスペースの役割に応じてスキルを選択することをお勧めします：

| ワークスタイプ | 推奨スキル |
|---|---|
| ドキュメントプロジェクト | ドキュメント整理、リンクチェック、翻訳支援 |
| コードプロジェクト | Git ワークフロー、テスト、コード生成、依存関係管理 |
| 運用プロジェクト | ログ分析、設定チェック、スクリプト実行 |
| コンテンツプロジェクト | 素材処理、Web ページ分析、フォーマット変換 |

すべてのスキルをすべてのワークスペースに同期しないでください。スキルが多いほど、AI が判断すべきコンテキストも増え、タスクの集中度が低下する可能性があります。

## 自動インジェクション

一部のスキルは自動インジェクションとして設定できます。自動インジェクションは、非常に安定していて、頻繁に必要な基本能力に適しています。例えば固定的なリポジトリコラボレーションフローやチーム規範などです。

自動インジェクション使用時は以下に従ってください：

- 確実に長期的に必要とされるスキルのみインジェクション
- 複数のスキルが互いに矛盾する操作要件を出さないようにする
- スキルの説明は簡潔で明確に保つ
- スキル更新後は数回のタスクパフォーマンスを観察

Claude Code がタスクで関係のないスキルを参照している場合は、自動インジェクションの範囲が広すぎるかを確認してください。

## 使用のヒント

- **タスク単位で蓄積**：「完全さ」のためにスキルを作成せず、高頻度タスクを優先的に蓄積
- **少数精鋭**：1つのワークスペースには本当に関連するスキルのみ同期
- **まず試してから展開**：新しいスキルはまず単一ワークスペースで検証し、その後さらに多くのワークスペースに同期
- **定期的なクリーンアップ**：長期間未使用、説明が古くなった、依存関係が無効なスキルは速やかに削除
- **セキュリティに注意**：スキルにアカウントパスワード、アクセストークン、内部機密アドレスを保存しない

## よくある質問

### スキルを同期済みなのに、タスクで使用されないのはなぜ？

スキルは強制実行項目ではありません。Claude Code はタスクに基づいて使用するかどうかを判断します。より安定的にトリガーしたい場合は、スキル名と説明を最適化して用途を明確にしてください。

### スキルをプラグインとして使用できますか？

混用はお勧めしません。プラグインは Nekro Agent のランタイム機能で、チャット、コマンド、ツール、システム統合を対象としています。スキルは Claude Code のタスク補助説明で、主にワークスペース内のコード、ファイル、自動化タスクを対象としています。

### スキルを削除すると過去のタスクに影響しますか？

削除後、新しいタスクでそのスキルを使用できなくなります。過去のタスクレコードは通常変更されませんが、その後同じ種類のタスクを続ける場合は、スキルを再選択または再同期する必要がある場合があります。
